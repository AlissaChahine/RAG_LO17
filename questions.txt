* The final refusal node also improves safety: if the final answer is still not grounded in the retrieved documents, the system refuses to answer instead of returning an unverified generation.

* However, HyDE can still introduce noise because it may generate hypothetical content that is not fully reliable. A possible improvement would be to make the HyDE prompt more restrictive, for example by asking for search keywords instead of a full hypothetical answer.

* Another remaining issue is answer completeness. Some answers can be grounded but still incomplete. For example, an answer may mention only part of the recipe or omit an important step.

* A possible improvement would be to make the answer grader stricter, so it checks not only whether the answer is relevant, but also whether it fully answers the question.

Updated pipeline structure:

1. First, the system separates the frontend input into the persona_prompt and the real user_question.
   → The persona_prompt is used only to control the final answer style.
   → The user_question is used for retrieval, multi-query, HyDE, and grading.

2. The system first performs standard retrieval using the original user_question.

3. The retrieved documents are evaluated by grade_documents to check whether they are sufficiently relevant.

4. If the documents are relevant enough:
   → The system generates an answer.
   → It checks whether the answer is grounded in the retrieved documents.
   → It checks whether the answer actually answers the user_question.
   → If both checks pass, the final answer is returned.

5. If the documents are not relevant enough, or if the generated answer is not grounded / not useful:
   → The system activates multi-query.
   → It generates 3 reformulations of the user_question.
   → Each query retrieves a small number of documents.
   → The retrieved documents are merged and deduplicated.
   → The merged documents are evaluated again by grade_documents.

6. If the documents retrieved through multi-query are relevant enough:
   → The system generates an answer.
   → It checks again whether the answer is grounded and useful.
   → If the checks pass, the final answer is returned.

7. If multi-query is still insufficient:
   → The system activates HyDE.
   → It generates a hypothetical answer / dense query.
   → The HyDE query is used to retrieve documents again.
   → The retrieved documents are evaluated again.
   → The system generates a new answer.

8. If the final answer is still not grounded or does not properly answer the question:
   → The system does not return the unverified answer.
   → It returns a refusal answer instead.
